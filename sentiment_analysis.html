Foundation

What is Sentiment Analysis?

Sentiment Analysis (also called opinion mining) is the use of Natural Language Processing (NLP) to automatically detect and classify the emotional tone behind a piece of text — typically as Positive, Negative, or Neutral.

🍕 Analogy — The Pizza Review

Imagine a restaurant owner reading 500 customer reviews every day. It would take hours to decide whether each review is happy or unhappy. Sentiment Analysis is like hiring a super-fast reader who can read all 500 reviews in one second and hand you three piles: "Loved it", "Hated it", and "Meh".

Real-world example

Given the sentence: "The battery life is amazing but the camera is terrible."

A sentiment model would detect positive sentiment toward the battery and negative sentiment toward the camera — a concept called Aspect-Based Sentiment Analysis (ABSA).

Pre-processing

Cleaning the Text

Before any model can read your text, you need to clean and prepare it. Raw text is messy — full of HTML tags, slang, punctuation, and inconsistent casing.

🥗 Analogy — Washing vegetables

You wouldn't cook vegetables straight from the farm — you wash off dirt, peel the skin, and chop them. Pre-processing is exactly that for text: you strip the "dirt" (URLs, HTML), peel the skin (punctuation), and chop it up (tokenization).

Pre-processing steps with examples

# Input sentence
text = "I LOVED the movie!!! 😍 It was soooo great... http://review.com #mustwatch"

# Step 1 — Lowercase
text = text.lower()
# → "i loved the movie!!! 😍 it was soooo great... http://review.com #mustwatch"

# Step 2 — Remove URLs, HTML, special chars
text = re.sub(r'http\S+|#\S+', '', text)
# → "i loved the movie!!! 😍 it was soooo great..."

# Step 3 — Tokenize (split into words)
tokens = ['i', 'loved', 'the', 'movie', 'it', 'was', 'soooo', 'great']

# Step 4 — Remove stopwords ('i', 'the', 'it', 'was')
tokens = ['loved', 'movie', 'soooo', 'great']

# Step 5 — Stemming / Lemmatization
# 'loved' → 'love'   'soooo' → 'so'   'great' → 'great'
tokens = ['love', 'movie', 'so', 'great']

Vectorization

Converting Words to Numbers

Machine learning models only understand numbers — not words. Vectorization converts text into numerical representations the model can process.

🎵 Analogy — Music Notation

A musician can't "feed music to a computer" directly — they encode it as sheet music (numbers: pitch, duration, volume). Vectorization is the sheet music for text: it translates the "music of language" into numbers a model can learn from.

Three popular approaches

── 1. Bag of Words (BoW) ────────────────────────────────────────
sentence = "I love this film"
vocabulary = {"I":0, "love":1, "this":2, "film":3, "hate":4}
vector    = [1, 1, 1, 1, 0]   # count of each word
# ✅ Simple  ❌ No word order  ❌ "not good" == "good not"

── 2. TF-IDF ────────────────────────────────────────────────────
# TF-IDF = Term Frequency × Inverse Document Frequency
# Rare words ("disgusting") get higher weight than common ones ("the")
tfidf["love"]       = 0.61   # appears often across docs — lower weight
tfidf["disgusting"] = 0.92   # appears rarely — higher weight (more signal)

── 3. Word Embeddings (Word2Vec / GloVe / BERT) ─────────────────
# Each word becomes a dense vector in high-dimensional space
# Similar meanings → similar vectors
embed["happy"]  = [0.82, −0.15, 0.44, ...]  # 300-dim vector
embed["joyful"] = [0.79, −0.18, 0.41, ...]  # very similar!
embed["sad"]    = [−0.71, 0.22, −0.38, ...] # very different

Modelling

Choosing a Model

There are three broad families of sentiment models, each with different trade-offs between simplicity and accuracy.

Option A — Lexicon / Rule-based (no training needed)

📖 Analogy — A Dictionary of Emotions

Imagine a dictionary where every word has a happiness score. "love" = +2, "hate" = −3, "okay" = 0. You add up all the scores in a sentence and decide: positive sum = positive sentiment. That's VADER — the most famous rule-based model.

from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer

analyzer = SentimentIntensityAnalyzer()
scores = analyzer.polarity_scores("The food was absolutely AMAZING!!!")
# → {'neg': 0.0, 'neu': 0.127, 'pos': 0.873, 'compound': 0.8622}
# Compound ≥ 0.05 → Positive ✅

Option B — Classical ML (Logistic Regression / Naive Bayes)

from sklearn.linear_model import LogisticRegression
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer(max_features=5000)
X_train   = vectorizer.fit_transform(texts)    # convert text → numbers
y_train   = [1, 0, 2, ...]                       # 0=neg, 1=pos, 2=neu

model = LogisticRegression()
model.fit(X_train, y_train)                      # train!

prediction = model.predict(vectorizer.transform(["I love it!"]))
# → [1]  (Positive)

Option C — Transformer (BERT / RoBERTa) — State of the Art

from transformers import pipeline

sentiment = pipeline("sentiment-analysis",
                      model="distilbert-base-uncased-finetuned-sst-2-english")

result = sentiment("Despite the delay, the service was wonderful!")
# → [{'label': 'POSITIVE', 'score': 0.9987}]

Evaluation

How Do We Measure Success?

A model that says "Positive" for everything would be 33% accurate on a balanced 3-class dataset — useless! We use four key metrics.

🩺 Analogy — COVID Test Accuracy

A COVID test that calls everyone negative is 99% "accurate" in a healthy population — but it's dangerously wrong. Precision and Recall catch this: you need the test to correctly catch the sick people (Recall) AND not wrongly flag healthy people (Precision).

MetricFormulaWhat it measures
Accuracy Correct / Total Overall correctness
Precision TP / (TP + FP) Of all predicted Positive, how many were actually Positive?
Recall TP / (TP + FN) Of all actual Positive, how many did we catch?
F1-Score 2 × (P × R) / (P + R) Harmonic mean — balances Precision and Recall 
from sklearn.metrics import classification_report

y_true = ['pos', 'neg', 'neu', 'pos', 'neg']
y_pred = ['pos', 'neg', 'pos', 'pos', 'neg']

print(classification_report(y_true, y_pred))
#               precision  recall  f1-score  support
# neg                1.00    1.00      1.00        2
# neu                0.00    0.00      0.00        1
# pos                0.67    1.00      0.80        2
# accuracy                             0.80        5

Challenges

Why is it Hard?

Human language is wildly nuanced. Here are the trickiest problems even state-of-the-art models struggle with:

ChallengeExampleWhy it's tricky
Sarcasm "Oh great, another meeting. Just what I needed." "great" is positive but the intent is negative
Negation "This is not bad at all." Double negation creates positive meaning
Domain shift "This stock is sick!" (slang) "sick" = amazing in Gen-Z, negative in medicine
Mixed sentiment "Great screen, terrible battery." Positive and negative about different aspects
Implicit sentiment "The queue moved for 3 hours." No explicit opinion word — but clearly negative
